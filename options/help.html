Rules

Pattern can be a Regular Expression.

The order is important. Rules may change state and are composable.

  • Changing state: If you have two rules and the first replaces foo with bar and the second replaces foo with bam, then the second rule has nothing to match, because all occurrences of foo have been replaced already.
  • Composability: Likewise, if your first rule substitutes foo with bar and your second rule replaces bar with bam, you could change that to one rule that replaces foo with bam.

Profiles

Profiles are small json files that define patterns. I prepared some profiles so you can activate them easily and don't have to write them yourself:

  • Trump to Drumpf
  • Cruz to Some Canadian Asshole (as requested)
  • Cyber to Bullshit
  • Cloud to Butt
  • Grammar Nazi

Contributions

You can contribute to the profiles:
  • fork the repository
  • if you introduce a new profile, create a new branch
  • write your commit message in present tense
  • issue a pull request
I am looking forward to get your feedback and suggestions, but please remember that this is my first extension and obviously an experiment. :)
